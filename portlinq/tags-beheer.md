# Tags beheren (PortlinQ-beheerder)

Deze gids is voor de **PortlinQ-beheerder** - de rol die de gecontroleerde tag-lijst voor de hele PortlinQ-dataspace bijhoudt. Tags houden API's en apps in de catalogus vindbaar en consistent.

Alleen de PortlinQ-beheerder mag de globale tag-lijst wijzigen; organisaties kunnen alleen tags uit die lijst toewijzen.

## Huidige tags

Op dit moment zijn de volgende tags beschikbaar:

| Tag | Betekenis |
| --- | --- |
| `port` | Haven - geeft aan welke diensten bij een specifieke haven horen |
| `shorepower` | Walstroom - dienst voor walstroomvoorziening |
| `visit` | Scheepsbezoek - dienst rond het bezoek van een schip aan de haven |
| `vessel` | Vaartuig - hiermee kun je filteren welke organisatie een vessel-dienst aanbiedt |

## Wat je kunt doen

| Mogelijkheid | Beschrijving |
| --- | --- |
| Tags aanmaken | Nieuwe tags toevoegen aan de gecontroleerde lijst |
| Tags hernoemen | Naamgeving aanpassen zonder de koppeling met systemen te verliezen |
| Tags verwijderen | Tags uit de lijst verwijderen wanneer de taxonomie wijzigt |
| Tags loskoppelen | Een toegewezen tag van een specifiek systeem verwijderen, bijvoorbeeld bij een foutieve toewijzing |

## Werkwijze

1. Open de Admin Portal en ga naar de tags-beheerpagina
2. Maak nieuwe tags aan voor consistente labeling in de catalogus
3. Hernoem of verwijder bestaande tags wanneer de taxonomie moet veranderen
4. Open eventueel een systeem-detailpagina om een foutief toegewezen tag te verwijderen

De lijst is dataspace-breed. Data Service Providers kunnen alleen tags toewijzen die al in deze lijst staan - zie [Tags toevoegen aan je diensten](tags-toevoegen-diensten.md).

## Richtlijnen voor tag-namen

- Gebruik korte, stabiele, herbruikbare namen
- Hanteer een naamconventie (bijvoorbeeld enkelvoud en lowercase IDs)
- Controleer regelmatig op duplicaten met een andere schrijfwijze
